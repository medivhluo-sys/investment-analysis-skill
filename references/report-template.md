# 格式和附录规范参考

本文件不再作为默认内容结构模板。个人投资者报告的内容结构以 `individual-investor-report-template.md` 为准。本文件仅保留可复用的格式、Source Card 附录、QA 附录和 PDF 渲染规范。

## 可选前端元素

- 术语表：所有专业名词首次出现需解释。
- KPI 仪表盘：只在关键指标足够可靠时使用，固定列数。
- 关于本报告：素材基础、额外工作、核心假设、局限性和标记体系。
- 核心观点与后续行动：CP 条件、假设表和后续工作建议。

## 决策标准表（可选）

以投资人设定的最低年化收益率为基准：

| 关键变量 | 可投区间 | 当前值 | 判断 |
|---------|---------|--------|------|
| （核心定价变量） | ≥ XX | 当前市场价 | ✅/⚠️/❌ |
| （核心成本变量） | ≤ XX | 当前市场价 | ✅/⚠️/❌ |
| （产能/利用率） | ≥ XX% | 预计值 | ✅/⚠️/❌ |
| （关键前置条件） | 已满足 | 当前状态 | ✅/⚠️/❌ |

## 后续工作三段式

### 一、后续尽调（投资决策前）
- 按周排期，每项标注目的（验证哪条 CP 条件）
- 区分「可联网验证」vs「需线下获取」

### 二、投资协议关键条款
- CP 条件 / 差补保护线 / 创始人锁定 / 信息权 / 反稀释 / 退出权

### 三、投后管理
- 分阶段：建设期 → 爬坡期 → 稳定运营 → 退出准备
- 每阶段含：频率 + 核心关注点 + 预警信号

## 格式规范

- 标题不用 emoji
- 正文 Georgia 衬线 + 中文宋体，标题 sans-serif
- 段落 3 句以上拆 bullet
- 表格表头大写，列宽手动指定
- callout 用全边框不用左侧彩色条
- KPI 卡片固定列数，数字带 `white-space: nowrap`
- PDF 用 Playwright，`display_header_footer=False`


## 来源和 QA 附录要求

- 参考来源表必须包含 Source Card 编号、证据等级、URL/文件路径、访问日期和本次用途。
- 对关键结论，报告正文应引用 Source Card 编号，而不是只列 URL。
- QA 门禁结果按 `references/qa-checklist.md` 的表格格式放入附录。
- 若存在高影响失败项，封面或“关于本报告”中必须标注“审阅版/待补充版”。
