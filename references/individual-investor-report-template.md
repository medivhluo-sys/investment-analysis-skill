# 个人投资者投前判断报告模板

本模板用于给投资人本人阅读。报告应短、直、结论先行，重点回答：现有数据能不能支撑判断，是否值得继续推进，需要补什么资料，收益测算依赖哪些未验证假设。

## 报告结构

1. 封面
   - 项目名称
   - 标的主体
   - 报告版本：初步判断版 / 待补充版 / 核验版
   - 数据截止日期

2. 一页结论
   - 结论：可推进 / 有条件关注 / 暂缓判断 / 不建议
   - 三句话说明理由
   - 最大 5 个数据缺口
   - 最大 5 个红旗
   - 下一步动作

3. 交易和标的概览
   - 投资金额、估值、股权比例或收益权
   - 交易结构和资金用途
   - 公司实际业务、客户、产品和阶段

4. 资料边界和证据等级
   - 收到哪些材料
   - 哪些是原件、草稿、管理层口径、访谈、公开来源
   - Source Card 摘要

5. 前置门禁结果
   - 八项扫描结果
   - 能否开工
   - 结论上限

6. 公开数据核验摘要
   - 已验证
   - 部分验证
   - 无法验证
   - 与公司口径冲突

7. 经营和财务判断
   - 收入真实性
   - 毛利和成本
   - 现金流和回款
   - 产能、订单、客户或项目进度

8. 收益测算
   - 输入假设表
   - Python 重算结果
   - 基准、保守、乐观情景
   - 对关键变量的敏感性
   - 未验证假设如何影响 IRR/MOIC 或回款

9. 红旗和投资者动作
   - P0/P1/P2 红旗登记表
   - 对估值、条款、前置条件和投后跟踪的影响

10. 信息缺口传导表
    - 每个缺口影响哪个判断
    - 需要补充什么
    - 补不到时结论降级到哪里

11. 下一步资料清单
    - 必须补
    - 重要但可后补
    - 可通过公开数据继续核验

12. QA 附录
    - 数据一致性
    - 来源等级
    - Python 重算
    - 结论降级
    - HTML/PDF 渲染

## 一页结论格式

| 项目 | 内容 |
|------|------|
| 当前结论 | 可推进 / 有条件关注 / 暂缓判断 / 不建议 |
| 最主要理由 | 1-3 条 |
| 最大不确定性 | 1-5 条 |
| 关键补证 | 1-5 项 |
| 收益测算状态 | 已重算 / 示意测算 / 不足以测算 |
| 下一步动作 | 补资料、访谈、重算、降价、加条款、停止 |

## 决策语言

| 情形 | 推荐写法 |
|------|----------|
| 数据足够 | “在当前证据和模型下，项目初步具备推进价值，但仍需完成以下投决前置事项。” |
| 数据部分缺失 | “项目存在可继续关注的投资逻辑，但核心假设仍依赖管理层口径，当前只能形成有条件关注结论。” |
| 数据不足 | “现有资料不足以支持正面投资判断，建议先补充以下资料后再决定是否继续。” |
| 红旗明显 | “该事项直接影响投资逻辑或退出可行性，在修复或解释前不建议推进。” |

## 禁止写法

- “确定可投”
- “收益有保障”
- “回购保证退出”
- “公司前景广阔，风险可控”
- “管理层预计”后直接给确定性收益结论
