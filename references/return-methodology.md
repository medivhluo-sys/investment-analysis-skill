# 收益率测算方法

## 核心原则

1. **必须先搞清楚分配规则**：年度 vs 退出是否走不同的瀑布
2. **必须先明确身份**：GP/LP/跟投，carry 归属不同
3. **不能直接引用材料中的收益率**，必须独立走一遍链条

## 分配规则核查

从基金/SPV 条款中提取：

| 分配类型 | 规则 | 是否走瀑布 |
|---------|------|-----------|
| 年度分配 | "扣除管理费后按实缴出资比例分配" | 通常不走 |
| 清算退出 | "返还本金→优先回报→GP carry→LP 按比例" | 走瀑布 |

常见陷阱：
- PPT 中"投资收益率 114%"可能是含 GP carry 的综合回报，不是纯 LP 的收益率
- 管理费（通常 1-2%/年）直接从基金资产扣，PPT 可能未扣除

## 分红链条（逐步穿透）

```
① 项目公司净利润（所得税后）
② 扣 10% 法定公积
③ 再扣 X% 经营留存 → 可分配利润
④ → 持股公司（× 持股比例）
⑤ → 基金（× 基金持持股公司比例）
⑥ → 我方（× 我方占基金比例，年度仅扣管理费按比例）
⑦ 退出时：盈余公积返还 + 回购 → 走瀑布（5.5%优先 + GP carry 20%）
```

## 管理费处理

- 管理费 = 基金规模 × 年费率
- 年度分红先扣管理费再分配
- 退出时清算瀑布也先扣管理费

## 三情景验证

PPT 的保守/中性/乐观假设不能直接引用：
- 中性：用外部市场数据交叉验证（价格是否接近当前市价？）
- 保守：变动幅度是否有历史先例？
- 乐观：是否与行业预测一致？

标注每项假设的合理性：基本合理 / 偏保守 / 偏乐观 / 需澄清


## 输出要求

收益测算附录必须包含：

1. 输入假设表：字段、数值、单位、来源、证据等级。
2. 分红链条穿透表：从项目公司到我方份额逐级计算。
3. 瀑布分配表：年度分配和退出分配分别列示。
4. Python 重算结果：IRR、MOIC、DSCR、差补、回购、管理费、carry。
5. 敏感性表：至少覆盖核心价格、销量/产能利用率、毛利率、退出倍数、退出时间。
6. 结论约束：若关键假设无 S/A 或双 B 来源支撑，不得输出“已验证收益率”，只能输出“模型测算结果，待验证”。
