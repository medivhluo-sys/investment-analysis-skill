# 公开数据核验剧本

本文件用于最大化公开数据的价值。公开数据不能替代完整尽调，但可以验证主体、风险、客户线索、行业口径、可比公司、价格和退出路径。

## 核验顺序

1. 确认主体和股权：官方工商、年报、变更、分支机构、关联公司。
2. 扫描负面风险：诉讼、执行、失信、行政处罚、环保、安全、税务、监管。
3. 验证业务线索：客户、招投标、中标、官网案例、新闻、产品备案、许可。
4. 验证技术或资质：专利、商标、软著、认证、标准、许可证。
5. 验证行业和价格：统计、主管部门、行业协会、上市公司公告、可比公司年报。
6. 验证退出和估值：可比上市公司、并购案例、IPO 规则、锁定期、回购方能力。

## Source Card

每个公开来源必须记录：

| 字段 | 内容 |
|------|------|
| Source Card 编号 | S001、S002 递增 |
| 来源名称 | 官方名称或文件名称 |
| URL / 文件路径 | 可追溯链接或本地文件路径 |
| 发布/主管机构 | 机构全称 |
| 发布时间/访问日期 | 动态网页必须写访问日期 |
| 证据等级 | S/A/B/C/D |
| 可验证字段 | 本次支持或反驳什么事实 |
| 局限性 | 滞后、口径不一致、样本不足、非官方等 |
| 影响的判断 | 收入、成本、估值、退出、合规、团队等 |

## 交叉验证逻辑

| 公司说法 | 公开数据核验动作 | 常见冲突 | 影响 |
|----------|------------------|----------|------|
| 收入快速增长 | 查客户线索、招投标、公开案例、可比公司行业增速 | 客户公开信息少、订单金额不匹配 | 收入真实性和增长质量降级 |
| 毛利率高于同行 | 查上市公司毛利、原材料价格、产品定位 | 毛利远高于同行但无技术或客户证据 | 利润和估值下调 |
| 技术壁垒强 | 查专利、软著、标准、认证、竞品参数 | 专利非核心、权属不在标的 | 技术壁垒待验证 |
| 客户质量高 | 查客户官网、招投标、新闻、上市公司披露 | 客户不存在公开采购或案例 | 客户真实性待确认 |
| 政策支持 | 查主管部门政策、补贴公示、准入规则 | 政策为宏观鼓励，非公司具体资质 | 不得作为确定收益依据 |
| IPO 或并购退出 | 查上市规则、可比案例、买方并购历史 | 行业退出稀少、锁定期或合规障碍 | 退出路径降级 |
| 回购/差补可靠 | 查回购方工商、负债、诉讼、资产和现金流线索 | 回购方轻资产或有执行风险 | 回购只能作为风险控制线索 |

## 公开数据不足时

如果未发现权威公开来源，不得用低质量来源强行填补。报告必须写：

- 未发现可免费获取的权威公开来源。
- 当前证据等级和局限性。
- 需要线下获取的资料。
- 对投资判断的影响和结论降级结果。

## 输出格式

公开核验摘要至少包含：

| 判断事项 | 公司口径 | 公开来源 | 证据等级 | 核验结果 | 冲突/缺口 | 结论影响 |
