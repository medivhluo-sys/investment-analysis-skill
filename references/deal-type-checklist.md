# 轻量交易类型判断

本文件用于第零层前置判断。目标不是做机构级交易适配器，而是先识别交易经济实质，避免用错误口径判断收益和风险。

## 使用规则

- 先判断用户实际买到的是什么：股权、债权化回购、老股、基金/SPV 份额、项目收益权或混合结构。
- 若结构混合，按最主要的现金回收来源分类。
- 若收益主要来自回购、差补、固定收益或担保，优先按偿付能力和可执行性判断，不把 IRR 当作唯一结论。

## 分类表

| 交易结构 | 关键问题 | 核心数据需求 | 收益判断基准 |
|----------|----------|--------------|--------------|
| 直接股权 | 公司是否能持续赚钱，未来能否退出 | 收入、利润、现金流、股权结构、退出路径 | IRR / MOIC |
| 明股实债或回购型 | 回购方是否有钱，回购是否可执行 | 回购协议、回购方资产负债表、现金流、担保、诉讼、抵质押 | 偿付能力优先于 IRR |
| Pre-IPO 老股 | 上市概率、时间、锁定期和估值折价是否合理 | 辅导备案、中介进场、历史财务合规、股权清晰、上市障碍、锁定期 | 估值折价 + 退出时间 |
| 基金/SPV 份额 | 底层资产是什么，费用和瀑布吃掉多少收益 | LPA/合伙协议、管理费、carry、优先回报、分配瀑布、底层资产资料 | 穿透后净 IRR |
| 项目收益权 | 项目现金流是否覆盖本金和收益 | 项目合同、收款权、验收、回款、成本、债务、DSCR、期限错配 | DSCR + 回款期限 |
| 混合结构 | 真实风险落在哪一层 | 全部交易文件、资金流、权利顺位、担保和退出安排 | 按最弱现金回收环节判断 |

## 结论约束

- 交易结构不清楚时，不得做收益率结论。
- 明股实债或回购型项目中，回购方偿付能力未验证时，不得写“下行有保障”。
- 基金/SPV 份额中，未取得管理费、carry、优先回报和分配瀑布时，只能做毛收益示意，不能写净收益。
- 项目收益权中，未取得项目合同、验收和回款安排时，不得把项目预测现金流当作可回收现金流。
