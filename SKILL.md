# 投资项目分析报告

**产出：完整的 HTML+PDF 投资分析报告**（含收益率测算、情景分析、风险矩阵、尽调清单、QA 验证）。

不是只出尽调清单。如果用户只需要尽调清单，调用 `private-equity:dd-checklist`。

基于公开信息（无需 API 接口），适用于非上市企业或项目结构。

## 触发条件

用户提供材料 + 意图是**做投资判断**（不是只出清单）。关键词：投资分析、能不能投、收益率测算、投资决策、看看这个项目、分析这个标的、帮忙判断一下。

## 前置：信息缺口扫描

收到项目材料后，**第一步不是开始分析，而是对照 8 项信息清单做缺口扫描**：

1. 交易结构 — 股权架构图 + 基金/SPV 条款 + 我方角色
2. 财务数据 — 近 3 年营收净利 + 项目盈利预测
3. 核心假设依据 — 盈利预测中关键假设的来源
4. 退出路径 — 方式和大致时间
5. 关键前置条件 — 融资关闭的前提（批文/贷款/认证）
6. 实控人 & 团队 — 背景和持股
7. 竞争格局 — 行业位置和主要竞对
8. 已有尽调材料

**缺了什么、有什么替代，先明确说出来**，再决定分析能到多深。
详见 `references/info-checklist.md`

## 分析流程（5 阶段）

### 阶段 1：素材提取
- PPT/PDF → `python-pptx` 或 `markitdown` 提取全文
- 图片 → OCR 或逐张描述
- 聊天记录 → 提取关键数据点
- 所有原始数据标注来源（页码/行号）

### 阶段 2：外部交叉核验
对材料中**每一项关键声明**做外部验证：
- 启动 3-4 路并行 Web 搜索 Agent（市场/竞对/技术/政策）
- 标注：🟢 完全验证 / 🟡 部分验证 / 🔴 无法验证
- 对每个 🔴 给出下一步验证路径
- 验证结果写入附录

### 阶段 3：独立财务核算
**绝不允许心算。** 必须 Python 独立重算：
- 资产负债表勾稽、利润表变动、现金流勾稽
- 弹性分析（收入拆分到具体产品线，不能用一个比例）
- 盈亏平衡（拆分固定/可变成本）
- DSCR（用满产年的实际还本付息额）
- 分红链条（逐步穿透，不能用一个系数）

### 阶段 4：收益测算
- **必须先搞清楚分配规则**：年度 vs 退出是否走不同的瀑布
- **必须先明确身份**：GP/LP/跟投
- 管理费、GP carry 必须计入
- 三情景（保守/中性/乐观）的假设不能直接引用材料，需独立验证合理性
- 详见 `references/return-methodology.md`

### 阶段 5：报告输出 & QA
- HTML 格式，自包含，可离线分发
- 结论前置（总体结论 + 决策标准 + CP 条件 + 后续行动）
- 启动独立 Agent 做 QA（非同一 Agent 自检）
- QA 失败时手动 Python 重算
- PDF 用 Playwright 导出（`display_header_footer=False` 避免乱码）

## 报告结构

详见 `references/report-template.md`。核心原则：

1. 结论和行动在前（投资人打开即见）
2. 收益率必须有方法说明
3. 决策标准表（以预期年化收益率为基准的变量可投区间）
4. 后续工作三段式：尽调 → 协议 → 投后
5. 正文在后（前结论后展开）
6. 附录含全量验证表

## 格式规范

- 标题不用 emoji
- 正文 Georgia 衬线体 + 中文宋体
- 段落内 3 句以上拆 bullet
- 专业术语首次出现加中文解释
- KPI 固定列数（不用 auto-fit）
- 表格列宽手动指定
- PDF 页眉空白，页码居右

## 重要注意事项

1. **财务数字必须 Python 重算**，不能心算
2. **收益率不能直接引用材料**，必须独立走一遍链条
3. **情景假设必须外部验证**，不能照搬 PPT
4. **分配瀑布必须读原文**（年度 vs 退出分别看）
5. **一个数字不能在报告里出现两个版本**——改一处，全局搜替换
6. **差补触发条件的数学必须精确**——不能"大概"
7. 券商研报（信达/中邮/华金/华源）+ 金联创 + Argus + IATA 是外部验证核心来源
8. 可比上市公司/已并购案例必须找对标估值
